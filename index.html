    Atividade Prática: Simulação de Infraestrutura em Nuvem com CloudSim

    Linux | Java JDK 21 | Apache NetBeans 15 | CloudSim 7.0.1

    Ver Roteiro Baixar PDF

    1. Introdução

    O que é Cloud Computing?

    Computação em nuvem é o modelo que permite acesso sob demanda a recursos computacionais (servidores, armazenamento, bancos de dados) pela internet, pagando apenas pelo que usa.

    Por que usar o CloudSim?

    CloudSim é um framework de simulação em Java que permite testar hipóteses de infraestrutura em nuvem sem custos de hardware e avaliar riscos antes da implantação real.

    Objetivo: Simular um Datacenter com Host e executar Cloudlets (tarefas)

    Importância da Simulação

    CloudSim permite testar escalabilidade e provisionamento dinâmico sem custos operacionais reais.

    Benefícios

    • ✅ Redução de Riscos

    • ✅ Escalabilidade
    • ✅ Provisionamento Dinâmico
    Conclusão Final: Simulação com CloudSim é fundamental para projetar infraestruturas em nuvem eficientes.

    5. Código Fonte

    
package redes;

import org.cloudbus.cloudsim.cloudlets.Cloudlet;
import org.cloudbus.cloudsim.cloudlets.CloudletSimple;
import org.cloudbus.cloudsim.core.CloudSim;
import org.cloudbus.cloudsim.datacenters.Datacenter;
import org.cloudbus.cloudsim.datacenters.DatacenterSimple;
import org.cloudbus.cloudsim.brokers.DatacenterBroker;
import org.cloudbus.cloudsim.brokers.DatacenterBrokerSimple;
import org.cloudbus.cloudsim.hosts.Host;
import org.cloudbus.cloudsim.hosts.HostSimple;
import org.cloudbus.cloudsim.resources.Pe;
import org.cloudbus.cloudsim.resources.PeSimple;
import org.cloudbus.cloudsim.schedulers.vm.VmSchedulerTimeShared;
import org.cloudbus.cloudsim.utilizationmodels.UtilizationModelFull;
import org.cloudbus.cloudsim.vms.Vm;
import org.cloudbus.cloudsim.vms.VmSimple;
import java.util.*;

public class Redes {
    public static void main(String[] args) {
        CloudSim cloudsim = new CloudSim();
        List<Host> hosts = new ArrayList<>();
        List<Pe> peList = new ArrayList<>();
        peList.add(new PeSimple(1000));
        peList.add(new PeSimple(1000));
        Host host = new HostSimple(8192, 100000, 10000);
        host.getCpuProvisioner().add(peList);
        host.setVmScheduler(new VmSchedulerTimeShared());
        hosts.add(host);
        Datacenter datacenter = new DatacenterSimple(cloudsim, hosts);
        datacenter.setName("Datacenter_SP");
        DatacenterBroker broker = new DatacenterBrokerSimple(cloudsim);
        List<Vm> vms = new ArrayList<>();
        Vm vm1 = new VmSimple(1024, 1);
        vm1.setMips(500);
        vms.add(vm1);
        broker.submitVmList(vms);
        List<Cloudlet> cloudlets = new ArrayList<>();
        Cloudlet c1 = new CloudletSimple(10000, 1);
        c1.setUtilizationModel(new UtilizationModelFull());
        cloudlets.add(c1);
        broker.submitCloudletList(cloudlets);
        cloudsim.start();
        System.out.println("✅ Simulação concluída!");
    }
}
    Explicação: O código cria um Datacenter com 1 Host (2 CPUs), uma VM e uma tarefa de 10k instruções.

    6. Questionário AV1 - Computação em Nuvem

    Modelos de Serviço | Elasticidade | Virtualização

    Questão 1

    "Com base nas características, benefícios e desafios da computação em nuvem, assinale a alternativa correta:"

    Resposta Correta: Alternativa D
    "O provisionamento sob demanda e a elasticidade rápida são viabilizados pelas tecnologias de virtualização utilizadas na computação em nuvem."
    Explicação detalhada: A virtualização é a tecnologia fundamental que permite que um único servidor físico rode múltiplas máquinas virtuais isoladas. É ela que possibilita o provisionamento sob demanda (criar recursos quando precisar) e a elasticidade rápida (aumentar ou diminuir recursos em minutos). Sem virtualização, a nuvem não existiria como conhecemos hoje.
    Alternativas incorretas:
    • Alternativa A: SaaS não dá controle sobre infraestrutura - o usuário só usa o software pronto (ex: Gmail, Netflix).
    • Alternativa B: Elasticidade é capacidade de ajustar recursos dinamicamente, não sobre cobrança fixa contratada.
    • Alternativa C: IaaS dá controle sobre infraestrutura (VMs, rede, storage), mas o usuário ainda gerencia SO e aplicações.
    • Alternativa E: A nuvem tem grandes vantagens econômicas: pague pelo que usar, sem investimento inicial em hardware.
    Caso de Uso: Uma startup de e-commerce que tem picos de vendas na Black Friday. Com virtualização e elasticidade, ela pode automaticamente aumentar servidores durante o pico (ex: 50 VMs) e reduzir depois (ex: 5 VMs), pagando apenas pelo que usou. Sem isso, teria que comprar servidores físicos caros que ficariam ociosos a maior parte do ano.
    Armazenamento | Big Data | AWS | Azure | GCP

    Questão 2

    "Assinale a alternativa que apresenta uma associação correta entre o tipo de serviço e sua principal finalidade."

    Resposta Correta: Alternativa E
    "O Google BigQuery é uma solução serverless para análise de dados, adequada para aplicações de inteligência de negócios."
    Explicação detalhada: O Google BigQuery é um data warehouse serverless que permite analisar terabytes de dados em segundos usando SQL. É amplamente usado para Business Intelligence (BI), dashboards e análises de grandes volumes de dados. Diferente de bancos tradicionais, ele escala automaticamente e cobra apenas pelo que é processado.
    Alternativas incorretas:
    • Alternativa A: Armazenamento em blocos (ex: AWS EBS) é para discos virtuais de VMs. Dados não estruturados via API é armazenamento de objetos (ex: AWS S3).
    • Alternativa B: Amazon Kinesis é para processamento de dados em tempo real (streaming), não para armazenamento persistente de arquivos.
    • Alternativa C: Azure Blob Storage é armazenamento de objetos, não banco de dados relacional com SQL.
    • Alternativa D: Amazon EFS é armazenamento de arquivos compartilhado, não "disco virtual" (isso seria EBS).
    Caso de Uso: Uma empresa de varejo com 10 milhões de vendas por dia usa o BigQuery para analisar padrões de compra em tempo real. Em segundos, o BigQuery processa dados que levariam horas em bancos tradicionais, permitindo à empresa ajustar promoções instantaneamente.
    Modelos de Implantação | Pública | Privada | Híbrida

    Questão 3

    "Assinale a alternativa correta quanto aos modelos de implantação de computação em nuvem."

    Resposta Correta: Alternativa C
    "A nuvem híbrida permite o uso combinado de nuvem pública e privada, sendo útil para aplicações com partes sensíveis que exigem segurança e outras que demandam escalabilidade."
    Explicação detalhada: A nuvem híbrida é o melhor dos dois mundos: dados sensíveis ficam na nuvem privada (mais segura, controlada), enquanto aplicações que precisam de escalabilidade usam a nuvem pública (AWS, Azure, GCP). É o modelo mais adotado por grandes empresas que precisam equilibrar segurança e custo.
    Alternativas incorretas:
    • Alternativa A: Nuvem pública é compartilhada entre múltiplas organizações, não é exclusiva de uma.
    • Alternativa B: A definição está incompleta e confusa. Nem toda aplicação híbrida precisa ficar totalmente isolada.
    • Alternativa D: Nuvem privada NÃO oferece escalabilidade ilimitada (tem limites físicos do hardware) e geralmente é mais cara.
    • Alternativa E: Nuvem comunitária é compartilhada entre organizações com interesses comuns (ex: bancos), não é pública com fins comerciais.
    Caso de Uso: Um banco mantém os dados de clientes (dados sensíveis) em sua nuvem privada por questões de compliance (LGPD). Mas durante o horário de pico (ex: Black Friday), ele utiliza a nuvem pública da AWS para processar transações, escalando automaticamente. Isso é nuvem híbrida.
    DevOps | IoT | Multimídia | Geoespacial

    Questão 4

    "Analise as afirmativas sobre serviços e soluções em nuvem para desenvolvimento de software, multimídia e IoT."

    Resposta Correta: Alternativa D
    "Apenas as afirmativas III, IV e V estão corretas."
    Análise detalhada das afirmativas:
    ❌ I (Incorreta): Azure Test Plans existe, mas é para gestão de testes manuais/exploratórios, NÃO para automação de testes (isso seria Azure DevOps Test Plans).
    ❌ II (Incorreta): AWS Elemental MediaConvert é para codificação de vídeo sob demanda (arquivos gravados). Para vídeo ao vivo, seria AWS Elemental MediaLive.
    ✅ III (Correta): Cloud IoT Core (Google) gerencia dispositivos IoT e coleta dados de sensores em tempo real.
    ✅ IV (Correta): AWS CodePipeline automatiza CI/CD (Continuous Integration/Continuous Deployment).
    ✅ V (Correta): Azure Maps fornece APIs geoespaciais para rastreamento, rotas, geolocalização.
    Caso de Uso: Uma empresa de logística usa Azure Maps para rastrear frotas em tempo real, AWS CodePipeline para atualizar automaticamente o app de entregas quando o código muda, e Google Cloud IoT Core para coletar dados de sensores de temperatura dos caminhões refrigerados.
    Armazenamento | NoSQL | Big Data | Bancos de Dados

    Questão 5

    "Analise as afirmativas sobre armazenamento em nuvem, bancos de dados em nuvem e Big Data."

    Resposta Correta: Alternativa A
    "Apenas as afirmativas III, IV e V estão corretas."
    Análise detalhada das afirmativas:
    ❌ I (Incorreta): Armazenamento transitório (ephemeral) é volátil - dados são perdidos quando a VM é reiniciada. O correto para persistência é armazenamento persistente.
    ❌ II (Incorreta): NoSQL é para dados não estruturados/sem esquema rígido. Para integridade referencial e estrutura rígida, usa-se bancos relacionais (SQL).
    ✅ III (Correta): Armazenamento hospedado é o armazenamento principal do cliente (ex: AWS S3, Azure Blob), focado em desempenho e disponibilidade.
    ✅ IV (Correta): Soluções de Big Data em nuvem se beneficiam da elasticidade e escalabilidade para processar grandes volumes de dados.
    ✅ V (Correta): Bancos de dados em nuvem oferecem backup automático, alta disponibilidade, tolerância a falhas e pagamento por uso.
    Caso de Uso: Uma rede social usa NoSQL (ex: DynamoDB) para armazenar posts e comentários (dados não estruturados), Big Data na nuvem (ex: EMR) para analisar tendências de engajamento, e bancos gerenciados (ex: RDS) para dados de usuários com backup automático e alta disponibilidade.

    Glossário Técnico

    Modelos de Serviço
    • IaaS (Infrastructure as a Service): Aluguel de infraestrutura (VMs, rede, storage). Ex: AWS EC2, Google Compute Engine.
    • PaaS (Platform as a Service): Plataforma de desenvolvimento pronta. Ex: Google App Engine, Heroku.
    • SaaS (Software as a Service): Software pronto pela internet. Ex: Gmail, Netflix, Salesforce.
    Modelos de Implantação
    • Nuvem Pública: Recursos compartilhados pela internet, pago por uso.
    • Nuvem Privada: Recursos exclusivos de uma organização, maior controle e segurança.
    • Nuvem Híbrida: Combinação de pública + privada, melhor dos dois mundos.
    • Nuvem Comunitária: Compartilhada entre organizações com interesses comuns.
    Tipos de Armazenamento
    • Armazenamento em Blocos: Discos virtuais para VMs (ex: AWS EBS, Azure Disk).
    • Armazenamento de Objetos: Dados não estruturados via API (ex: AWS S3, Azure Blob).
    • Armazenamento de Arquivos: Sistemas de arquivos compartilhados (ex: AWS EFS, Azure Files).
    Conceitos Fundamentais
    • Elasticidade: Capacidade de ajustar recursos automaticamente conforme demanda.
    • Virtualização: Tecnologia que permite múltiplas VMs em um único servidor físico.
    • Serverless: Executa código sem gerenciar servidores (ex: AWS Lambda).
    • Big Data: Processamento de grandes volumes de dados (volume, velocidade, variedade).
    • NoSQL: Bancos de dados não relacionais para dados não estruturados.

    Microserviços | Arquitetura Monolítica | Escalabilidade

    Questão 1

    "Considerando as principais características e vantagens da arquitetura de microserviços em comparação à arquitetura monolítica, analise as afirmações..."

    Resposta Correta: Alternativa C
    "Apenas as afirmativas I, II e IV estão corretas."
    Análise detalhada:
    ✅ I (Correta): Cada microserviço é um serviço independente e altamente coeso.
    ✅ II (Correta): Microserviços permitem replicar seletivamente apenas os módulos com maior demanda.
    ❌ III (Incorreta): Em arquitetura monolítica, qualquer atualização exige reinicialização completa.
    ✅ IV (Correta): Microserviços favorecem contêineres e bancos distintos por serviço.
    ❌ V (Incorreta): Em monolítico, não é possível replicar módulos independentemente.
    Caso de Uso: Netflix migrou de monolítico para microserviços, permitindo escalar apenas o serviço de catálogo durante picos de acesso, sem afetar o serviço de recomendações.
    Segurança | IAM | Criptografia | SSO | Hardened VMs

    Questão 2

    "Considerando os mecanismos de segurança aplicados a serviços em nuvem e os princípios do gerenciamento de riscos, analise as afirmativas..."

    Resposta Correta: Alternativa A (ou E)
    "Apenas as afirmativas II, III e IV estão corretas."
    Análise detalhada:
    ❌ I (Incorreta): Criptografia também pode garantir integridade (hash) e autenticidade (assinaturas).
    ✅ II (Correta): IAM é essencial para controle de privilégios e autenticação.
    ✅ III (Correta): SSO facilita gestão segura de credenciais entre múltiplos provedores.
    ✅ IV (Correta): Hardened VMs são configuradas com políticas rígidas de segurança.
    ❌ V (Incorreta): Gerenciamento de riscos é contínuo, não apenas no momento da contratação.
    Caso de Uso: Uma empresa que usa AWS, Azure e Google Cloud implementa SSO para que funcionários acessem todos os serviços com uma única senha, enquanto IAM define quem pode criar ou deletar recursos.
    Migração | Re-host | Re-platform | Re-factor | IAM | SSO

    Questão 3

    "Com base no processo de migração de aplicações para a nuvem e nos mecanismos de controle de acesso, analise as afirmativas..."

    Resposta Correta: Alternativa D
    "Apenas as afirmativas I, III e IV estão corretas."
    Análise detalhada:
    ✅ I (Correta): Re-host (Lift & Shift) é rápido mas não aproveita recursos nativos da nuvem.
    ❌ II (Incorreta): Re-platform adapta parcialmente; quem redesenha profundamente é Re-factor.
    ✅ III (Correta): IAM é fundamental para controle de acesso em nuvem.
    ✅ IV (Correta): Re-factor (redesenho) exige maior esforço mas maximiza benefícios da nuvem.
    ❌ V (Incorreta): SSO não é dispensável em nuvem privada; continua importante para segurança.
    Caso de Uso: Uma empresa migrou seu sistema de RH com Re-host (rápido, mas sem otimização) e seu sistema de vendas com Re-factor (totalmente redesenado para usar Lambda e DynamoDB), maximizando escala e reduzindo custos.
    Precificação | On-Demand | Reserved | TCO | SaaS

    Questão 4

    "Sobre os modelos de precificação e as práticas de otimização de custos na computação em nuvem, analise as afirmações..."

    Resposta Correta: Alternativa A
    "Apenas as afirmativas I, III, IV e V estão corretas."
    Análise detalhada:
    ✅ I (Correta): On-Demand cobra pelo uso real, alocação dinâmica.
    ❌ II (Incorreta): Reserva de longo prazo tem preços MAIS BAIXOS que on-demand (é a vantagem).
    ✅ III (Correta): Desligar recursos não utilizados reduz custos significativamente.
    ✅ IV (Correta): TCO considera custos operacionais (OPEX) e de capital (CAPEX).
    ✅ V (Correta): SaaS usa modelo de assinatura periódica.
    Caso de Uso: Uma empresa reserva instâncias por 3 anos (Reserved Instance) pagando 60% menos que on-demand, e desliga ambientes de desenvolvimento aos finais de semana, economizando 40% adicionais.
    Docker | Kubernetes | Contêineres | Virtualização | Hypervisor

    Questão 5

    "Com base nos conceitos apresentados sobre virtualização e containerização de aplicações, analise as afirmativas..."

    Resposta Correta: Alternativa C
    "Apenas as afirmativas I, III e IV estão corretas."
    Análise detalhada:
    ✅ I (Correta): Contêineres são mais leves por não incluírem SO próprio.
    ❌ II (Incorreta): Isolamento completo com SO distintos é característica de VMs, não contêineres.
    ✅ III (Correta): Docker empacota aplicação com dependências, garantindo consistência.
    ✅ IV (Correta): Kubernetes orquestra contêineres em clusters.
    ❌ V (Incorreta): Hypervisores (XenServer, ESXi) são para VMs, não para orquestração de contêineres.
    Caso de Uso: Spotify usa Docker para empacotar microserviços e Kubernetes para gerenciar milhares de contêineres, permitindo deploy contínuo e escala automática durante picos de streaming.

    Glossário Técnico - Parte 2

    Arquitetura
    • Microserviços: Arquitetura que divide aplicações em serviços pequenos e independentes
    • Monolítico: Aplicação única onde todos os módulos são acoplados
    • Re-host: Migração "lift and shift" sem alterações
    • Re-platform: Adaptação parcial para serviços de nuvem
    • Re-factor: Redesenho completo da arquitetura
    Segurança
    • IAM: Gerenciamento de Identidade e Acesso
    • SSO: Single Sign-On - autenticação unificada
    • Hardened VM: Máquina virtual com configurações de segurança rigorosas
    Precificação
    • On-Demand: Pagamento pelo uso real, sem compromisso
    • Reserved: Reserva antecipada com desconto
    • TCO: Custo Total de Propriedade
    Containerização
    • Docker: Plataforma de empacotamento de contêineres
    • Kubernetes: Orquestrador de contêineres
    • Hypervisor: Camada que gerencia máquinas virtuais
